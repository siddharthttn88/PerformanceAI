## Key Characteristics of This Format:

1. **Summary Table Layout:**
   - Row 1-2: Blank for spacing
   - Row 3: Load information with TPS calculation
   - Row 4: Test timing information with ramp-up
   - Row 5: Pass/Fail status
   - Row 6: Detailed comment section (multi-line)
   - Row 7: Blank separator
   - Row 8: Data table headers

2. **Summary Section Structure:**
   - Uses pipe (|) separators between columns
   - Column 1: Label (Load, Test Time, Test Status, Comment)
   - Column 2: Main value
   - Column 3: Additional info (Total TPS on service, RampUp time)

3. **Data Table:**
   - Uses pipe (|) separators
   - 11 columns total
   - Includes percentile data (90th, 95th)
   - Numeric precision maintained (2 decimal places for response times)

4. **Comment Section Details:**
   - Multi-line format with infrastructure details
   - Includes: Server count, pod configurations, feature flags
   - Documents environment-specific settings

5. **Test Status Determination:**
   - Based on response times (P95 > 1000ms indicates issues)
   - Based on error rates
   - Based on infrastructure stability

6. **Spacing & Formatting:**
   - 6 blank lines between test results (for separation)
   - Consistent column alignment
   - Headers in Row 8 always

---

## Differences from template.txt Format:

1. **TPS Display:**
   - Subscriber format: Shows target TPS in Load row, actual TPS separately
   - Original: Single TPS value

2. **Comment Section:**
   - Subscriber format: More detailed infrastructure configuration
   - Original: Brief summary

3. **Additional Columns:**
   - RampUp information in separate column on Row 4

4. **Status Row:**
   - Explicit "Test Status" row instead of combined
